📦 405实验室 Data Query 报账系统 - 打包与部署说明
本系统采用“前后台双系统”架构，基于纯原生 Python + Tkinter + SQLite3 开发。为了适应实验室各种办公电脑的环境，系统采用 PyInstaller 单文件（One-File）模式进行打包，最终交付物为绿色免安装版的独立桌面程序。

一、 打包环境准备
在执行打包前，请确保开发机已安装 Python 环境，并且安装了 pyinstaller 打包工具。
打开终端（终端/CMD/PowerShell），运行以下命令安装：

Bash
pip install pyinstaller
(注：本项目未引入 pandas、matplotlib 等重型外部库，原生打包即可，无需配置复杂的虚拟环境)

二、 核心打包命令
系统分为前台录入端（main.py）和后台管理端（admin_backend.py），需分别进行打包。请在项目代码所在的根目录（包含 logo.ico 的目录）下打开终端，依次执行以下两条命令：

1. 打包前台录入端：

Bash
pyinstaller -F -w -i logo.ico main.py
2. 打包后台管理端：

Bash
pyinstaller -F -w -i logo.ico admin_backend.py
💡 打包参数解析：
-F (或 --onefile)：将程序及所有 Python 底层依赖压缩成一个独立的 .exe 文件。

-w (或 --noconsole / --windowed)：隐藏底层的黑色命令行窗口，让程序启动时直接显示图形化界面，提升软件的正式感。

-i logo.ico：为打包后的 .exe 文件绑定你设计的专属实验室图标。

三、 打包产物与目录清理
打包命令执行完成后，你的项目目录下会自动生成几个新的文件夹和文件：

dist 文件夹：【核心】 里面存放着打包成功的 main.exe 和 admin_backend.exe。

build 文件夹：打包过程产生的临时编译文件（打包完成后可直接删除）。

.spec 文件：PyInstaller 的配置文件（main.spec 和 admin_backend.spec，如果不需要修改高级打包配置，也可保留或删除）。

四、 最终交付与部署结构（绿色免安装）
由于系统采用 SQLite 本地相对路径数据库（lab_billing_system.db），本软件无需任何安装程序。

在向实验室电脑部署时，只需在电脑桌面或 D 盘新建一个名为 “405实验室财务报账系统” 的文件夹，并保持以下标准目录结构即可直接双击运行：

Plaintext

五、 版本更新与数据保留原则
未来如果你修复了 Bug 或增加了新功能（例如新增了模块），需要给实验室更新软件，请严格遵循以下原则：

只替换 .exe：只需要将你在开发机上重新打包好的新版 admin_backend.exe 或 main.exe 拷贝到实验室电脑的文件夹中，覆盖旧的 .exe。

绝对不要覆盖 .db 文件：千万不要把你开发测试用的 lab_billing_system.db 拷贝过去覆盖掉实验室电脑上的同名文件，否则会导致实验室历史报账数据全部丢失！

数据自适应：新版 .exe 启动后，会自动读取同级目录下原有的 .db 文件，历史数据无缝衔接。

六、 常见问题与注意事项 (FAQ)
Q1：打包后双击 .exe 杀毒软件（如 360/电脑管家）报错或拦截怎么办？

原因：这是 PyInstaller 单文件打包（释放临时文件到系统 C 盘 Temp 目录）的普遍现象，容易被杀软误判为病毒特征。

解决办法：请在杀毒软件中选择“添加信任”或“允许程序运行”。作为内部独立开发的工具，完全安全。

Q2：打开软件发现数据全是空的？

原因：由于采用相对路径，这说明 .exe 所在的同级目录下没有找到 lab_billing_system.db。

解决办法：请检查是否不小心把 .exe 单独移动到了别处。必须确保前台程序、后台程序和数据库文件始终“待在同一个文件夹”里。
