# 开源部署流程

> 本文档指导你将「实验室报账系统」完整开源到 GitHub，供他人使用和贡献。

---

## 📋 前置准备

| 项目 | 说明 |
|------|------|
| GitHub 账号 | 前往 [github.com](https://github.com) 注册 |
| Git 客户端 | [下载 Git](https://git-scm.com/) 并安装 |
| Python 3.10+ | 已安装并配置环境变量 |
| PyInstaller | 可选（仅打包时需要） |

---

## 🚀 部署步骤

### 第一步：创建 GitHub 仓库

1. 登录 GitHub，点击右上角 `+` → `New repository`
2. 填写仓库信息：

   ```
   Repository name: reimbursement-system
   Description: 实验室报账系统 - 高校报销管理工具
   Visibility: Public
   Initialize with: (均不勾选，我们手动推送)
   ```

3. 点击 `Create repository`

---

### 第二步：本地初始化 Git 仓库

打开命令行终端，进入开源包目录：

```bash
# 进入开源目录
cd F:\work2\报账软件bug修复\开源

# 初始化 Git
git init

# 创建 .gitignore 文件（见下方内容）
```

创建 `.gitignore` 文件：

```gitignore
# Python 缓存
__pycache__/
*.pyc
*.pyo
*.pyd
*.db

# PyInstaller 输出
build/
dist/
*.spec

# 系统文件
Thumbs.db
.DS_Store
*.log

# 数据库（用户本地数据，不应提交）
*.db
```

---

### 第三步：提交代码

```bash
# 添加所有文件到暂存区
git add .

# 查看文件状态（确认只包含需要的文件）
git status

# 创建首次提交
git commit -m "🎉 Initial commit: 实验室报账系统 v1.0

一个专为高校实验室设计的报销管理工具，支持日常报销单
和差旅费报销单的智能解析与数据管理。"
```

---

### 第四步：推送到 GitHub

```bash
# 添加远程仓库（替换 YOUR_USERNAME 为你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/reimbursement-system.git

# 推送到 main 分支
git push -u origin main
```

> 🔑 如果提示需要登录，GitHub 会弹出浏览器窗口进行 OAuth 认证，或使用 Personal Access Token。

---

### 第五步：创建 Release（可选，但推荐）

当代码稳定并准备发布可执行文件时：

1. 在 GitHub 仓库页面，点击右侧 `Releases` → `Create a new release`
2. 填写发布信息：

   ```
   Tag version: v1.0
   Release title: v1.0 - 初始开源发布版本
   Description:
   ## v1.0 (2026-05-29)
   
   ### 功能特性
   - 日常报销单与差旅费报销单智能解析
   - 发票明细表自动提取与金额合计
   - 人员信息管理
   - 数据入库与多维度查询导出
   - 批量设置发票所属人
   ```

3. 将 `dist/main.exe` 打包为 ZIP，上传到 Release 附件
4. 点击 `Publish release`

---

### 第六步：发布到其他平台（可选）

| 平台 | 建议方式 |
|------|----------|
| **Gitee / 码云** | 镜像 GitHub 仓库，适合国内用户访问 |
| **知乎/简书** | 写一篇项目介绍文章，引导用户到 GitHub |
| **实验室内部** | 提供 Release 中的 exe 直接下载 |

---

## 📦 发布清单核对

每次新版本发布前，核对以下事项：

- [ ] `main.py` 编译语法通过（`python -m py_compile main.py`）
- [ ] 数据库初始化可正常执行（删除旧 `.db` 文件后运行）
- [ ] PyInstaller 打包成功（`pyinstaller --onefile --noconsole --name main --icon=logo.ico main.py`）
- [ ] exe 文件可以正常启动和操作
- [ ] README.md 版本号已更新
- [ ] 所有敏感信息已脱敏（人员数据库默认使用张三/李四等测试数据）
- [ ] `.gitignore` 排除了 `.db`、`build/`、`dist/`、`__pycache__/` 等

## 🧪 持续维护建议

### Issue 管理

在 GitHub Issues 中打标签分类：

| 标签 | 用途 |
|------|------|
| `bug` | 程序错误报告 |
| `enhancement` | 功能建议 |
| `question` | 使用疑问 |
| `help wanted` | 寻求贡献者帮助 |

### Pull Request 流程

1. 贡献者 fork 仓库并提交 PR
2. 通过 GitHub Actions 自动运行语法检查
3. 维护者 review 代码后合并

### 版本号规范

遵循语义化版本：

```
v<主版本>.<次版本>.<修订号>

v1.0 → 主版本 1，次版本 0
```

- 主版本：不兼容的 API 变更
- 次版本：向下兼容的新功能
- 修订号：向下兼容的 bug 修复

## 📎 附录

### 常用 Git 命令速查

```bash
# 查看修改状态
git status

# 查看修改内容
git diff

# 暂存特定文件
git add main.py

# 提交
git commit -m "提交信息"

# 推送到远程
git push

# 拉取最新
git pull

# 查看提交历史
git log --oneline --graph
```

### 打包命令速查

```bash
# 安装 PyInstaller
pip install pyinstaller

# 打包为单文件 exe（无控制台窗口）
pyinstaller --onefile --noconsole --name 报账系统 --icon=logo.ico main.py
```
