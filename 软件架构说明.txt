一、 整体架构：前后台物理分离的“双系统”设计
系统采用了经典的单机 C/S 架构，通过在代码层面分离前台与后台，实现了权限与职能的解耦：

前台录入端 (main.py)：专注于高效率的数据采集。内置了文本信息的提取解析逻辑，配合原生 UI 实现发票录入、自动归类、明细扣款等基础操作，并能自动打包归档财务附件（ZIP）。

后台管理端 (admin_backend.py)：定位于高权限的数据治理、全局分析、容错校验和数据安全管控，是整个系统的数据中枢。

二、 核心技术栈与“极致轻量化”原则
纯原生 Python + Tkinter：坚决摒弃了臃肿的第三方数据处理或图表库（如 pandas, matplotlib 等），全部采用 Python 标准库实现。无论是复杂的 Excel 级数据网格，还是响应式数据大屏，均通过 Tkinter 原生控件及 Canvas 纯手工绘制。这使得最终打包的 .exe 文件体积极小，无外部依赖，能在各种配置的办公电脑上秒速启动。

本地 SQLite3 引擎：以单文件 lab_billing_system.db 作为核心存储引擎，无需额外部署配置服务器端数据库，极大地降低了实验室的软件分发和运维门槛。

三、 数据库设计 (关系型核心表)
数据底座遵循“主-从”结构设计，逻辑严密：

Reimbursement_Forms (主表)：记录宏观报账单据（如预约单号、总金额、经费号、报销类型等）。

Invoices (发票明细表)：与主表通过 form_order_id 形成外键关联，精确记录单张发票号与金额。

Personnel (人员花名册)：通过 personnel_id 与发票表关联，确保随时可以追踪每一笔发票资金的归属人（涵盖姓名、年级、学号、银行卡号等信息）。

四、 后台核心功能全景图 (6大模块)
后台系统目前已集成了六个强大的核心业务模块：

📊 报销数据管理：提供了 Excel 式的双击就地编辑体验，带有斑马纹 UI。内置“一键重新计算所有总金额”引擎，在导入历史数据后，能自动根据发票明细反向校准主表金额，确保账目分毫不差。

👥 实验室人员管理：灵活的人员花名册增删改查系统，支持一键生成空白行快速录入新进实验室的人员。

📥 数据导入导出 (双通道引擎)：支持财务单据和人员名册的独立 CSV 互导。具备极强的容错性，导入时利用 REPLACE INTO 机制防止主键冲突和重复覆盖，并附带控制台级别的实时滚动执行日志。

📈 可视化大屏：纯原生 Canvas 算法绘制的响应式看板。实时动态展示：经费分类占比（饼图）、人员报销Top 10贡献榜（柱状图）、各经费号累计资金消耗（柱状图）。

🧾 发票查验系统 (防重防漏)：利用 SQL 的 LEFT JOIN 联合查询，通过发票号码即可瞬间溯源查验。有效拦截重复发票，全方位展示该发票的归属单号、时间、经费号及录入人员，彻底封堵“一票多报”的财务漏洞。

💾 数据库备份与恢复：防范数据灾难的最后一道防线。支持系统自动快照与手动定点备份；在执行高危的“覆盖恢复”操作前，系统还会自动触发一次 failsafe 紧急快照，实现数据的双重保险。

五、 隐形的安全与容错底座
核心字段防篡改：所有数据网格中带有 🔒 图标的核心列（如底层ID、主预约单号），通过底层事件拦截了双击修改，严格维护了数据库的关系完整性。

强制类型转换保护：自研了 safe_float 拦截机制，防止用户误输入、空值或带逗号的非法字符导致整个系统崩溃。

中文字符排序重构：底层利用正则表达式重写了日期提取与解析逻辑，彻底解决了原生 Tkinter 在处理包含中文字符日期时，排序逻辑混乱的痛点。
